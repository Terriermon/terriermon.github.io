git revert的使用

会回退到指定Id的代码（本地和远程都会删掉之前的代码）

将当前回退的代码作为一个【新】的commitId提交

git diff的使用

  • 查看暂存区与最后提交的差异
  • 统计一下有哪些文件被改动
  • 查看当前目录和另外一个分支的差别
  • 比较两个历史版本之间的差异

第一部分: 显示修改的文件.

如:

“—“ ， 表示变动前的文件.
“+++” ， 表示变动后的文件.

第二部分: 修改文件中变动的位置.

如: @@ -1,7 +1,7 @@

  1. 文件变动的位置使用 @@起始，并以@@结束.
  2. -1中的负号，表示修改前的文件
  3. +1中的正号，表示修改后的文件
  4. -1,7表示修改之【前】的文件中，第1行到第7行
  5. +1,7表示修改之【后】的文件中，第1行到第7行
  6. -1,7 +1,7表示修改之【前】文件的第一行到第7行，变成了，修改之【后】文件的第1行到第7行.

git blame使用

  • 查看某个文件的每一部分代码，分别是谁提交的
  • 定位某个文件第1行到第3行，是谁修改提交的
